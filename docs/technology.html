GitChain: blockchain made simple

Our approach is pretty simple we consider any git repository as a "content-addressable-store" which can be distributed over a DHT layer. Immutability is provided by the native interlinking of commit objects via they hash (SHA1 or SHA256 depending how you've setup your repository; (we initially use IPFS to prove our concept).


a Git repository is already a ledger, by adding elliptic curve key pair you add authentication to git, allowing not only the option to sign commit but also to push branch via "pull request" to any public keys (the same way a blockchain user would send a token to a wallet).

Each git user is given a ed25519 key pair to authenticate their requests to the network
In order to have a truly decentralize repository (serverless) we add a pull request (inbound) queue for each public key, this allows any member to discover repositories over the DHT as well as to push a branch via sending a pull request to the recipient of their choice.
the recipient of a request has the option to pull (fetch and merge) their branch to their local work repository... Conflict are solved the same usual way with git.

Moreover if the objects pushed are of the type CRDT, we can guarantee always conflict-free merge making it operate as a consensus layer, making git a efficient blockchain. 

For our first test we augmented git with the following commands:

  • join : in order to join the network and make the repository discoverable
  • members : to discover other remotes (public-keys) for a given repository
  • notify: sending a pull requests to a public-key
  • prqueue: view the pending pull requests

Note: this is an early on going project, aiming to integrate blockchain capability to git.
